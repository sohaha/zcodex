# 回归检查清单

## 通用检查

- 验证相关回调、任务、异步链路是否仍然正常。
- 检查状态流转、重试逻辑和幂等性是否被影响。
- 当行为随环境变化时，确认配置优先级和覆盖关系仍然正确。

## 上游同步与合并前后

- 同步前先整理“分叉版独有功能/汉化”保留清单，不要只看冲突文件。
- 同步后逐项核对 `core` 的上下文重注入、turn `cwd` 配置重算、`reference_context_item` 保留是否仍在。
- 同步后逐项核对 `tui` 的 buddy 可见性/状态文案、approval overlay、plugins 弹窗、history_cell、`request_user_input` 是否仍为分叉版预期文案与行为。
- 如果是技能驱动的上游同步，先在独立 worktree 完成差异审计与回归，再合并回主工作树。
