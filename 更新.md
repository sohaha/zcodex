# 本轮同步补齐项

本文件记录本次 `openai/codex` 同步后，`tui_app_server` 相对旧 `tui` 的关键缺口，以及本轮直接实施的补齐范围。

## 背景判断

- 本次大规模 diff 主要来自上游自上次同步点之后的真实变更。
- 本地架构策略保持：
  - `tui` 行为参考 `tui_app_server`
  - `workspace` 尽量贴上游
  - `artifacts` 跟随上游移除，避免继续维护孤儿实现
  - `native-tldr`、`rtk` 等本地增强必须保留

## 已确认缺口

### 1. `/title` 终端标题配置入口丢失

- 旧 `tui` 仍有完整入口、预览、保存、取消回滚链路
- `tui_app_server` 中：
  - `TerminalTitleSetupView` 与相关 `AppEvent` 仍然存在
  - 终端标题渲染逻辑也仍然存在
  - 但 slash command 枚举和分发入口缺失
  - `App` 侧也未处理对应保存/预览/取消事件

结论：这是明显的“半接线”状态，应恢复。

### 2. custom prompts 会话级枚举未接通

- 旧 `tui` 在 `SessionConfigured` 后会发送 `ListCustomPrompts`
- 收到 `ListCustomPromptsResponse` 后，会把 prompts 注入 bottom pane/composer
- `tui_app_server` 当前直接忽略 `ListCustomPromptsResponse`

结论：`/prompts:*` 的解析代码虽然还在，但会话级数据源未接入，导致功能实质不可用，应恢复。

### 3. `/apps` 在 slash popup 中被错误过滤

- `tui_app_server` 运行时支持 `/apps`
- 但命令弹窗里额外把 `SlashCommand::Apps` 过滤掉了

结论：这是 UI 暴露层的回归，应删除这层过滤。

### 4. 记忆维护命令被退化为 stub

- 旧 `tui`：
  - `/debug-m-drop` -> `Op::DropMemories`
  - `/debug-m-update` -> `Op::UpdateMemories`
- `tui_app_server` 当前只提示“暂不支持”

结论：既然后端 `Op` 仍存在，就应直接恢复提交，不保留假 stub。

## 本轮实施范围

- 恢复 `/title` 在 `tui_app_server` 中的命令入口与事件处理
- 恢复 custom prompts 的枚举请求与响应接线
- 恢复 `/apps` 在 slash popup 中的可见性
- 恢复记忆维护命令直通 core `Op`

## 不在本轮扩展的内容

- 不新造一套偏离上游的 TUI 架构
- 不恢复 `artifacts`
- 不扩大到无关模块重构，只做缺口补齐与必要编译修复
