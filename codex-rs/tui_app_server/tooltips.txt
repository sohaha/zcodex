对话变长时可用 /compact 总结历史，释放上下文。
用 /new 开一个全新话题；上一段会话仍会保留在历史中。
发现异常时可用 /feedback 把日志发给维护者。
用 /model 可快速切换模型或推理强度。
用 /permissions 控制 Codex 何时向你请求确认。
运行 /review 可对当前改动做代码审查。
用 /skills 可列出可用 Skills，或让 Codex 使用某个 Skill。
用 /status 查看当前模型、审批设置和 Token 用量。
用 /statusline 配置状态栏显示哪些项目。
用 /fork 可把当前聊天派生到一个新线程。
用 /init 创建带项目专属说明的 AGENTS.md。
用 /mcp 列出已配置的 MCP 工具。
运行 `codex app` 可打开 Codex Desktop（macOS 需要时会自动安装）。
用 /personality 自定义 Codex 的沟通方式。
用 /rename 重命名线程，后续恢复线程时更容易区分。

你可以在 Codex 中用 `!` 执行任意 shell 命令（例如 `!ls`）
输入 / 打开命令弹窗；Tab 可自动补全 slash command。
当输入框为空时，按 Esc 可回退并编辑上一条消息；按 Enter 确认。
任务运行中按 Tab 可将消息加入队列；否则会立即发送（`!` 除外）。
按 Ctrl+V 粘贴图片，可把它附加到下一条消息。
运行 `codex resume` 可恢复之前的对话。
