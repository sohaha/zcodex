# tmux 窗口尺寸变更后渲染不全修复

## 背景
- 当前状态：`codex-cli` 的 TUI 在窗口尺寸变更后偶发渲染不全；用户确认关闭 buddy/宠物后仍可复现，说明不是 buddy 专属问题。
- 触发原因：在 `tmux` 下调整 pane/window 尺寸后更容易出现；静态排查显示 `codex-rs/tui/src/tui.rs` 中 `pending_viewport_area()` 会在 resize 时根据光标位置差值推断 inline viewport 偏移，该启发式在 tmux 下不稳定。
- 预期影响：修复后，TUI 在 tmux 下 resize 不应出现内容截断；非 tmux 终端也不应因同一路径产生同类错位。

## 目标
- 目标结果：将 resize 后 viewport 重定位改为稳定、可验证的逻辑，并补充针对该回归的测试。
- 完成定义（DoD）：在 `codex-tui` 中完成最小修复；新增覆盖“resize 时光标位置不可靠”的回归测试；相关 crate 测试通过；若行为变化影响现有快照则一并更新。
- 非目标：无意重做整个 TUI 布局系统；无意改动 buddy 功能、消息队列展示逻辑或无关终端特性。

## 范围
- 范围内：`codex-rs/tui/src/tui.rs` 的 resize / viewport 逻辑；必要时配套调整相关测试代码与快照。
- 范围外：`core`、`cli`、`app-server`、非 resize 触发的渲染问题、tmux 配置兼容层。

## 影响
- 受影响模块：`codex-rs/tui/src/tui.rs`、相关 TUI 测试模块。
- 受影响接口/命令：`codex` TUI 交互；无新增外部命令接口。
- 受影响数据/模式：无持久化数据或 schema 变更。
- 受影响用户界面/行为：inline viewport 在终端 resize 后的定位与重绘行为。

## 约束与依赖
- 约束（时间/兼容性/安全/性能/发布窗口等）：保持改动最小；优先通用修复而不是仅做 tmux 特判；避免引入新的闪烁或错误滚动；遵守仓库 TUI/测试规范。
- 外部依赖（系统/人员/数据/权限等）：依赖本地 `codex-tui` 测试环境；无需额外外部权限。

## 实施策略
- 总体方案：先确认现有 resize 路径中哪些状态在 tmux 下不可靠，再把 viewport 重定位改为更确定的计算方式；随后补回归测试覆盖“resize 发生但 cursor 反馈不可信”的场景。
- 关键决策：
  - 不再把光标位移启发式作为唯一依据。
  - 修复应面向所有终端通用路径，tmux 只是重点回归场景。
  - 先做最小可验证修复，再看是否需要扩展更多终端兼容处理。
- 明确不采用的方案（如有）：
  - 不仅靠 buddy/宠物开关规避问题。
  - 不通过增加隐藏重试或静默 fallback 掩盖状态错误。
  - 不先做大规模 TUI 重构。

## 阶段拆分
### 根因确认与修复设计
- 目标：锁定 resize 后 viewport 错位的具体状态链路，并确定最小修复方案。
- 交付物：可实施的修复设计，明确受影响函数与测试切入点。
- 完成条件：能够说明为什么 tmux 更易复现、为什么不是 buddy 根因，以及准备修改的具体代码路径。
- 依赖：现有静态分析结论。

### 实现与回归测试
- 目标：完成代码修复并补充测试。
- 交付物：代码改动、单元/集成测试、必要的快照更新。
- 完成条件：新增测试能覆盖 resize 相关回归，且实现通过对应 crate 验证。
- 依赖：根因确认与修复设计。

### 验证与收尾
- 目标：完成格式化、测试、自审与变更总结。
- 交付物：验证结果、风险说明、必要时的提交。
- 完成条件：满足仓库要求的最小验证链路，并可清楚说明已验证范围与剩余风险。
- 依赖：实现与回归测试。

## 测试与验证
- 核心验证：运行 `cargo nextest run -p codex-tui`，或在缺少 nextest 时运行 `cargo test -p codex-tui`。
- 必过检查：`just fmt`；若改动规模达到需要 lint 修复的程度，则运行 `just fix -p codex-tui`。
- 回归验证：覆盖“resize 后 cursor 位置不可靠/未变化”的测试场景；如涉及 UI 文本/布局变化，检查并接受对应 snapshot。
- 手动检查：在 tmux 中调整 pane/window 尺寸，观察 resize 后是否仍出现底部内容截断；必要时对比非 tmux 终端基本行为。
- 未执行的验证（如有）：无。

## 风险与缓解
- 关键风险：修复 viewport 锚定后可能影响非 tmux 终端原本可用的 resize 行为。
- 触发信号：现有相关测试失败、snapshot 大范围变化、普通终端 resize 后出现跳屏或滚动异常。
- 缓解措施：先补测试再收敛改动；仅修改 viewport 重定位最小路径；保留对非 tmux 的回归验证。
- 回滚/恢复方案（如需要）：若新逻辑引发更广泛回归，可回退到修改前版本并重新拆分为更小策略调整。
